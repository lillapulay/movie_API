Documentation

MyFlix will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

REST API Movie and User requests

Business Logic URL HTTP Method Request body data format Response body data format
1) Return a list of ALL movies to the user /movies GET None An array of JSON objects listing all movies.
2) Return data (title, director, genre) about a single movie by title to the user /movies/[title] GET None A JSON object holding data about a single movie by title. Example: 
                { 

                  id: '1', 

                  title: 'Forrest Gump', 

                  director: 'Robert Zemeckis', 

                  genre: 'Drama' 

                }
3) Return data about a genre (description) by name/title (e.g., “Thriller”) /genres/[name] GET None A JSON object holding data about a genre (description) by name or title. Example: 
                {

                  name: 'Drama',

                  description: 'Within film, television and radio,

                  drama is a genre of narrative fiction...'

                }
4) Return data about a director (bio, birth year, death year) by name /directors/[name] GET None A JSON object holding data about a single director. Example: 
                {

                  name: 'Robert Zemeckis',

                  bio: 'American director',

                  birth: '1952',

                  death:''

                }
5) Allow new users to register /users POST A JSON object holding data about the new user to register. Example: 
                {

                  "username" : "Sarah White",

                  "password" : "MyAccount10",

                  "email" : "sarah.white@gmail.com",

                  "birth" : "1995-06-04"

                  }
A JSON object holding data about the new user created. Example: 
                {

                  "username" : "Sarah White",

                  "password" : "MyAccount10",

                  "email" : "sarah.white@gmail.com",

                  "birth" : "1995-06-04",

                  "id": "0b6812dd-9a96-4039-91b1-70876861273a"

                  }
6) Allow users to update their user info (username, password, email, date of birth) /users/[username] PUT A JSON object holding user data to be updated. Example: 
                {

                  "username" : "Sarah Jane White",

                  "password" : "MyAccount10",

                  "email" : "sarah.white@gmail.com",

                  "birth" : "1995-06-04"

                  }
A text message indicating whether the user's information was succesfully updated.
7) Allow users to add a movie to their list of favorites /users/[username]
/favorites/[movieID] 		PUT None A text message indicating whether the movie was succesfully added to the list.
8) Allow users to remove a movie from their list of favorites /users/[username]
/favorites/[movieID] 		DELETE None A text message indicating whether the movie was succesfully removed from the list.
9) Allow existing users to deregister /users/[username]/ DELETE None A text message indicating whether the user was succesfully removed.
